# 🥗 개인 맞춤형 식단·운동 루틴 자동 설계 서비스
> **AI Vision과 동적 로직을 결합한 지능형 헬스케어 솔루션**
> 사진 한 장으로 식단을 분석하고, 개인의 일상 이벤트(회식, 야근 등)에 맞춰 운동 및 식단 루틴을 실시간으로 재구성합니다.
 
---
 
## 1. 프로젝트 개요 (Overview)
* **목적**: 수동 기록의 번거로움을 줄이고, 돌발 상황(이벤트)을 반영하지 못하는 기존 헬스케어 서비스의 한계를 개선.
* **핵심 타겟**: 체계적인 관리를 원하지만 정해진 루틴을 지키기 어려운 직장인 및 학생.
* **주요 성과**: 창업스쿨 AI·SW 융합콘텐츠 부문 **최종 성과 발표회 우수상** 시상.
 
---

* **YOLOv8 & EfficientNet**: 객체 탐지(ROI 추출) 후 이미지 임베딩 기술을 적용, 고차원 벡터 유사도 분석으로 150종 이상의 음식을 정밀 분류.

## 3. 주요 기능 (Key Features)
 
* **지능형 식단 인식**: 사진 촬영 시 AI 파이프라인이 즉시 영양 성분을 추출하고 DB와 매핑하여 기록.
 
* **적응형 루틴 조정 (Event-Driven)**: 회식·야근 등 이벤트 발생 시, 즉각적인 실패 처리 대신 정책 기반으로 영향을 분산하여 루틴을 자동 재조정합니다.
 
  ```
  이벤트 입력 (회식 · 야근 · 과식)
    → event_policy        : 이벤트 유형 분류 및 보정 정책 결정
    → horizon_engine      : 초과 칼로리를 2~3일 단위로 분산 계산
    → replan_orchestrator : 식단 + 운동 루틴 동시 재설계
    → DB 저장 · 사용자 화면에 즉시 반영
  ```
 
* **개인화 대시보드**: BMI 및 활동량 기반 목표 달성도를 시각화 리포트로 제공.

## 4. 프로젝트 구조 (System Architecture)
유지보수와 관심사 분리를 위해 약 100여 개의 모듈을 계층화하여 설계했습니다.
 
* **UI Layer**: 사용자 인터페이스 및 세션 상태 관리 (`ui_streamlit/`)
* **Service Layer**: 이벤트 대응 로직 및 영양 목표 계산 등 비즈니스 알고리즘 (`service/`)
* **Event Layer**: 이벤트 감지 · 정책 판단 · Horizon 기반 루틴 재계획 (`events/`)
* **Infrastructure Layer**: DB 연결 및 보안 설정 (`infra/`)
* **Data Layer**: 데이터 모델링 및 SQL 쿼리 관리 (`sql/`)
